# AI 초록 감별기(AI Abstract Detector) 개발 및 구현 계획서

본 계획서는 `proposal.txt`에 명시된 연구 목적과 `data_analysis.ipynb`에서 검증된 통계적 분석 결과를 바탕으로, 인간이 작성한 논문 초록과 AI(GPT-4o)가 생성한 초록을 정밀하게 판별하는 **AI 초록 감별기** 개발 프로젝트의 전 과정을 정의합니다.

---

## 1. 프로젝트 개요 및 목표
* **배경**: 생성형 AI(LLM)의 발전으로 학술적 글쓰기나 과제물 작성 시 표절 및 오남용 사례가 증가하고 있습니다. 이에 따라 텍스트의 구조적 무작위성과 문체 패턴을 학습하여 AI 저작물 여부를 식별하는 시스템이 필요합니다.
* **목표**: 컴퓨터공학/ICT 분야 논문 초록을 대상으로 인간 작성 원본과 AI 생성본을 높은 정확도(F1-Score 90% 이상)로 분류하는 머신러닝/딥러닝 기반 감별 시스템 구축.
* **활용 데이터셋**: 총 1,000개의 데이터 포인트 (`dbpia_computer_science.csv` 기반: Human 작성 초록 500건 + GPT-4o 생성 초록 500건 쌍)

---

## 2. 기반 데이터 분석 결과 요약 (핵심 탐지 자질)
감별기 모델의 Feature(피처)로 활용할 통계적 분석 결과는 다음과 같습니다.

| 자질 (Feature) | 인간 작성 초록 (Human) | AI 생성 초록 (AI) | 통계적 유의성 및 특징 | 감별기 활용 방식 |
| :--- | :---: | :---: | :--- | :--- |
| **Perplexity (PPL)** | **73.11** (높음) | **54.32** (낮음) | p < 0.05. 인간의 문장이 구조적으로 더 복잡하고 무작위성이 큼. | **핵심 수치 피처**: 낮은 PPL은 AI 생성물 확률을 높임. |
| **Type-Token Ratio (TTR)** | **0.8589** (낮음) | **0.9032** (높음) | p < 1e-40. AI가 단어의 기계적 반복을 극도로 회피하여 어휘 다양성이 높음. | **핵심 수치 피처**: 높은 TTR 지표 반영. |
| **그룹 내 코사인 유사도** | **0.5379** (낮음) | **0.5879** (높음) | AI 그룹 내 텍스트 균질성(Homogeneity)이 유의미하게 높음. | **문체 임베딩 피처**: AI 특유의 학술 템플릿 패턴 매핑. |

---

## 3. 감별기 아키텍처 및 구현 파이프라인

프로젝트는 데이터 전처리, 피처 추출, 모델 학습, 추론 인터페이스 개발의 4단계 파이프라인으로 진행됩니다.

```
[텍스트 입력] -> [전처리/토큰화] -> [피처 추출 (PPL, TTR, Embedding)] -> [앙상블 분류기] -> [AI 확률 결과 출력]
```

### 단계 1: 데이터 전처리 및 라벨링
* **라벨링 지정**: Human 초록 (`abstract` 컬럼) = `0`, AI 생성 초록 (`fake_abstract` 컬럼) = `1`로 정의하여 이진 분류(Binary Classification) 데이터셋 구축.
* **텍스트 정제**: 줄바꿈 기호(`
`), 중복 공백, 학술지 메타데이터 노이즈 제거.

### 단계 2: 피처 엔지니어링 (Feature Engineering)
텍스트로부터 모델에 입력할 다차원 피처를 추출합니다.
1. **수치형 언어 지표 (Statistical Features)**
   * **PPL 계산 모듈**: 분석에 사용된 `KoGPT2` 모델을 파이프라인에 이식하여, 입력 문장에 대한 Loss 기반 Perplexity 값을 실시간 계산 및 변수화.
   * **어휘 복잡도(TTR) 계산**: 형태소 분석기(KoNLPy 등)를 활용하여 형태소 단위의 고유 단어 비율(TTR) 산출.
   * **길이 지표**: 글자 수, 단어 수, 문장당 평균 단어 수 산출.
2. **시맨틱 문체 임베딩 (Semantic Features)**
   * `KR-SBERT` 모델을 활용하여 초록 전체를 768차원의 밀집 벡터(Dense Vector)로 변환. 이는 AI 특유의 정형화된 학술 템플릿(문체적 균질성)을 포착하는 역할을 합니다.

### 단계 3: 모델링 전략 (Modeling Strategy)
성능 극대화를 위해 두 가지 접근법을 결합한 앙상블(Ensemble) 형태를 취합니다.

* **Approach A: 하이브리드 Tabular 모델 (LightGBM / XGBoost)**
  * 추출된 수치형 지표(PPL, TTR, 길이)와 차원 축소(PCA)된 SBERT 임베딩 벡터를 결합하여 LightGBM 분류기에 입력합니다.
  * *장점*: 학습 및 추론 속도가 매우 빠르며, 어떤 피처가 판별에 결정적 기여를 했는지 파악하기 쉽습니다.
* **Approach B: End-to-End 언어 모델 Fine-tuning (KoELECTRA / RoBERTa)**
  * 텍스트 원문을 입력받아 AI 여부를 바로 분류하는 이진 분류 레이어를 사전 학습 언어 모델 위에 추가하여 미세조정 학습을 진행합니다.
  * *장점*: 수치적으로 잡히지 않는 미세한 문맥적 흐름과 부자연스러운 조사 사용 패턴까지 잡아낼 수 있습니다.

## 4. 모델 평가 및 검증 계획

* **데이터 분할**: 데이터셋 불균형을 방지하기 위해 Stratified 5-Fold Cross-Validation(교차 검증)을 적용하거나, Train/Val/Test 데이터를 7:1:2 비율로 엄격히 분리합니다.
* **주요 평가 지표**: 
  * **Accuracy (정확도)**: 전체 예측 성공률
  * **F1-Score (우선 지표)**: AI 초록 감별의 정밀도와 재현율을 균형 있게 평가하기 위한 지표
  * **ROC-AUC**: 모델의 클래스 분류 판별력 측정
* **설명 가능성 확보 (XAI)**:
  * Tabular 모델에 **SHAP(SHapley Additive exPlanations)** 알고리즘을 적용하여, 감별기가 특정 초록을 AI로 판단했을 때 PPL, TTR, 문체 벡터가 각각 얼마나 기여했는지 시각화 기능 제공.

## 5. 구현 일정 및 마일스톤

* **1주차: 데이터 모듈화 및 피처 추출 스크립트 작성**
  * Raw 텍스트에서 KoGPT2 PPL 및 TTR을 자동으로 추출하여 저장하는 데이터 파이프라인 완성.
* **2주차: 모델 학습 및 하이퍼파라미터 최적화**
  * LightGBM Baseline 구축 및 KoELECTRA Fine-tuning 진행. 두 모델의 예측 소프트맥스 확률 값을 블렌딩(Soft Voting)하여 최종 성능 확보.
* **3주차: 데모 시스템 구축 및 최종 평가**
  * `Streamlit` 기반의 웹 대시보드 UI 구현. 유저가 논문 초록을 복사-붙여넣기 하면 "AI 생성 확률: 87.5% (낮은 구조적 무작위성 및 정형화된 문체 감지)" 형태로 분석 리포트를 시각화하여 보여주는 프로토타입 완성.
